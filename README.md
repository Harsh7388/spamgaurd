# SpamGuard – Spam Email Detection Using Machine Learning

A full-stack web app that classifies email messages as **Spam** or **Not Spam** using
a trained scikit-learn pipeline, served via a Flask REST API and consumed by a React frontend.

## Project Structure

```
SpamGuard/
├── frontend/        React + Vite app (Home, Detect, About pages)
└── backend/          Flask API + ML training pipeline
    ├── app.py             Flask server (/predict, /health)
    ├── train_model.py     Trains NB, Logistic Regression, SVM and saves the best one
    ├── preprocess.py      Shared text cleaning (used by both training and inference)
    ├── dataset/sms.tsv    Labeled spam/ham dataset
    ├── spam_model.pkl     Saved best model (generated by train_model.py)
    ├── vectorizer.pkl     Saved TF-IDF vectorizer
    └── model_metadata.json
```

## Backend Setup

```bash
cd backend
pip install -r requirements.txt
python -c "import nltk; nltk.download('stopwords'); nltk.download('wordnet'); nltk.download('omw-1.4'); nltk.download('punkt'); nltk.download('punkt_tab')"

# Train the model (creates spam_model.pkl + vectorizer.pkl)
python train_model.py

# Run the API
python app.py
```
The API runs at `http://localhost:5000`.

### API

**POST** `/predict`
```json
{ "message": "Congratulations! You have won a free iPhone." }
```
Response:
```json
{
  "prediction": "Spam",
  "confidence": 97.8,
  "spam_probability": 97.8,
  "ham_probability": 2.2,
  "keywords": ["free", "won", "congratulations"],
  "explanation": "This email contains words such as ...",
  "model_used": "Support Vector Machine"
}
```

**GET** `/health` — returns model name and comparison metrics.

## Frontend Setup

```bash
cd frontend
npm install
npm run dev
```
The app runs at `http://localhost:5173` and expects the backend at `http://localhost:5000`
(configurable via `frontend/.env` → `VITE_API_URL`).

## ML Pipeline Summary

1. **Dataset**: a labeled spam/ham SMS/email-style dataset (5,572 messages).
2. **Preprocessing**: lowercasing, punctuation removal, stopword removal, tokenization, lemmatization (NLTK).
3. **Vectorization**: TF-IDF (unigrams + bigrams, 5,000 features).
4. **Models trained & compared**: Multinomial Naive Bayes, Logistic Regression, SVM (linear kernel).
5. **Model selection**: best F1-score model is auto-selected (SVM, ~98.3% accuracy in this run).
6. **Serving**: best model + vectorizer are saved with Joblib and loaded by the Flask API.

## Notes
- This project ships with a real, pre-trained model — no internet access is required at inference time.
- The dataset used here (SMS Spam Collection) is a standard, widely-used public spam/ham dataset; swap in any other labeled spam-email CSV/TSV by updating `train_model.py`'s data loading step.
